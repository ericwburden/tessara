# Tessara UI Direction

This document is retained as a compatibility pointer.

The canonical source for active Tessara UI decisions is now [ui-guidance.md](./ui-guidance.md).

Use [ui-guidance.md](./ui-guidance.md) for:

- product posture and naming
- brand, palette, theme, and shell behavior
- screen families and information architecture
- SSR, hydration, and lazy-loading guidance
- layout, component, state, and messaging standards
- transitional `Dataset -> Component -> Dashboard` UI constraints

`roadmap.md`, `requirements.md`, and `architecture.md` remain authoritative for delivery scope, product requirements, and system architecture.
