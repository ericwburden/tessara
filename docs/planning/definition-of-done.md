# Definition Of Done

## Purpose

Summarize Sprint 2F completion rules for Orpheum planning traceability.

## Completion Guidance

This file supports the `planning-traceability` check.

## Done Criteria

- Runtime status is visible to authorized operators.
- Materialization readiness and refresh state are visible where supported.
- Existing workflow-mediated response flows remain working.
- Scoped operators cannot see out-of-scope runtime/readiness data.
- Touched routes remain native SSR-owned and hydration/console-clean.
- Stable errors and tracing are in place for touched paths.
- Required Rust, Playwright, smoke, UAT, audit, and Orpheum checks are run or explicitly documented.
