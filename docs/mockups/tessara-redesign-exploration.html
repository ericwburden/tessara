Unified shell mockups

One shared Tessara space, shaped by permissions

These mockups remove any admin/user shell split. The app uses one shell, one navigation model, and one visual language. Routes, sidebar groups, actions, and support panels appear only when the user's permission set allows them.

Shown with admin:all dev-only hints omitted Sidebar footer owns account and theme Top bar is search plus quiet utilities

Home keeps the queue first and the hierarchy second
